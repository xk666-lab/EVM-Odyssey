# 什么是业务能力

> 💡 **业务能力 = 将技术转化为用户价值的能力**

---

## 定义

```
业务能力 = 理解用户需求 + 设计解决方案 + 创造商业价值

技术能力：
- 知道如何实现
- 能写出代码
- 代码可运行

业务能力：
- 知道为什么做
- 能设计方案
- 创造用户价值

区别：
技术专家 → 解决技术问题
业务专家 → 解决用户问题
```

---

## 为什么重要？

```
DeFi开发者的3个阶段：

阶段1：会写代码
- 能实现ERC20
- 能部署合约
- 月薪：$3K-5K

阶段2：懂技术设计
- 能设计架构
- 能优化Gas
- 月薪：$8K-15K

阶段3：懂业务设计  ← 稀缺！
- 能分析需求
- 能设计协议
- 能创造价值
- 月薪：$20K-50K+

业务能力是最稀缺的！
```

---

## 实例对比

**只懂技术：**

```solidity
// 实现了功能，但...
function swap(uint amountIn, address[] path) external {
    // ... swap逻辑
    pair.swap(amount0Out, amount1Out, to, "");
}
```

**懂技术+业务：**

```solidity
// 考虑了用户体验和安全
function swapExactTokensForTokens(
    uint amountIn,
    uint amountOutMin,  // ← 业务：滑点保护
    address[] calldata path,
    address to,
    uint deadline       // ← 业务：防MEV
) external ensure(deadline) {  // ← 业务：时间保护
    // 业务：计算预期输出
    amounts = getAmountsOut(amountIn, path);
    
    // 业务：保护用户
    require(amounts[last] >= amountOutMin);
    
    // ... swap逻辑
}
```

**差异：**
- 前者只考虑功能实现
- 后者考虑用户需求、安全、体验

## 如何培养？

```
1. 多问WHY
   - 为什么Uniswap要做Flash Swaps？
   - 为什么协议费默认关闭？

2. 站在用户角度
   - 如果我是LP，我关心什么？
   - 如果我是交易者，我需要什么？

3. 研究商业模式
   - Uniswap如何可持续？
   - Curve为什么成功？

4. 分析竞品
   - Uniswap vs Curve区别？
   - 各自的优势是什么？

5. 实践
   - 设计自己的协议
   - 从需求出发
   - 持续迭代
```
